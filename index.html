Enterprise Fintech · Hiring Assignment

Payment Platform Stabilisation

Stop the bleed. Restore enterprise trust. One quarter, zero disruption.

The problem

Peak traffic triggered DB lock contention → retry storms → settlement engine failures. Reconciliation mismatches weren't a reporting bug - they were a symptom of transaction data written out of chronological order. Ops team was manual middleware.

My approach

  • Telemetry audit to map failure rates by client, payment rail, error code
  • Separated structural errors (true mismatches) from timing errors (sync lag)
  • Ops time-motion study to baseline manual reconciliation hours
  • P0 prioritisation: highest impact, minimal engineering lift

Solutions shipped

  • Idempotent retry logic with exponential backoff + jitter
  • 30-min reconciliation delay buffer post peak windows
  • Exception triage dashboard (Retool) - Ops out of Excel

Key insight

"Operational stabilisation over platform transformation - stop the bleed for internal teams, restore predictability for enterprise clients, within a single quarter, zero disruption."

Consumer AI · PM Talentathon

Swiggy AI - Rider Support Resolution

From raising tickets to resolving issues within the same interaction.

The problem

Riders with payout questions got generic AI responses, no resolution, and lost trust in the platform. The system was optimised for process, not outcomes. Four distinct gaps: Explanation, Context, Action, Ownership - each eroding trust at a different stage.

Core diagnosis

Current AI was built for queries, not situations. It matched keywords and returned predefined responses. It had no decision layer - it could not validate, only redirect. Wrong financial explanations destroyed trust faster than any other failure mode.

Solution: Context-Aware Resolution System

  • Detects rider intent; uses existing orders, earnings, incentive context
  • Explains "why" using real data - no more confusion about earnings
  • Confidence threshold logic: AI resolves above 90%, human escalates below
  • P0 focus: Earnings clarity (highest cost of misinformation)

Rollout strategy

Phase 1: MVP pilot in 1 high-volume city (Pune/Bangalore), Hindi + English + local language. Phase 2: 3-5 cities, edge case handling. Phase 3: Pan-India, voice (vernacular), proactive nudges.

B2B SaaS · PM Talentathon

Knowledge Management Platform Adoption

From knowledge library to GPS for expertise.

The problem

65% of registered users abandoned the knowledge base after 30 days. Support teams still received repetitive questions that already had documented answers. The platform was a library when users needed a GPS - passive storage vs active guidance.

Root cause diagnosis

  • Discovery failures - agents couldn't find answers fast enough
  • Workflow friction - required leaving tools to search
  • Decaying trust - wrong or stale answers collapsed confidence
  • Authoring tax - creating knowledge was too expensive

Three-pillar strategy

  • Workflow-native delivery - CRM/ticket plugins, intent-based snippets, 1-click insert
  • Knowledge capture pipeline - AI drafts from closed tickets, gap detection
  • Verified knowledge engine - SME approval queue, freshness signals, confidence scoring

Prioritisation logic

RICE framework, weighted for time-to-value. Q1: Embedded Answer Intelligence + Trust Signals (remove search friction). Q2: Resolution → Knowledge Pipeline (automate creation). First build the GPS interface, then the GPS learning engine.

HRTech · PM Case Challenge

Payroll Workflow Reliability

Prevent errors earlier. Make workflows transparent. Stop regressions. 4-6 weeks.

The problem

Rising support tickets, manual Excel workarounds, and churn risk - all symptoms of broken system trust. Payroll is a financial accuracy product. When HR teams lose trust, they build parallel systems and evaluate competitors.

Four failure modes

  • Validation vacuum - errors discovered only post-processing
  • Black box workflow - no explainability, no auditability
  • Release regressions - weekly shipping broke edge cases unnoticed
  • Edge-case blind spots - no guidance for mid-cycle, retroactive scenarios

Data-first approach

  • Support ticket clustering by stage: input, calculation, payout
  • Workflow drop-off funnel via Mixpanel/Amplitude
  • Manual correction audit - every spreadsheet = a missing product rule
  • Release correlation analysis - overlay tickets against release notes

Solutions (no rebuild)

  • Wks 1-2: Pre-flight validation layer + payroll health summary
  • Wks 2-3: Variance & summary dashboard (black box → audit receipt)
  • Wks 3-6: Edge-case patches + regression test suite + feature flags

Industrial SaaS · PM Hiring Assignment

Condition Monitoring SaaS Platform

Signal-agnostic predictive intelligence for industrial maintenance.

Domain expertise applied

Identified the technical ambiguity in the brief between 4-20mA process telemetry and Motor Current Signature Analysis - a distinction that shapes the entire product architecture. Defined the platform around MCSA (high-frequency harmonic analysis) as the defensible differentiation.

Platform architecture

  • Signal-agnostic ingestion: Vibration, MCSA, Thermal, Process via MQTT/OPC-UA
  • Multi-tenant design: JWT isolation, RBAC, dynamic schema for new signal types
  • Three-phase roadmap: Mechanical → Electrical → Unified AI with RUL models

Market framing

$23B industrial predictive maintenance market by 2027. 47% of unplanned downtime avoidable. 3-5× ROI on predictive vs reactive. Buyer: Plant Manager/VP Maintenance (downtime cost, OEE). User: Reliability Engineer (early fault detection, less alert fatigue).

Security & compliance

AES-256 encryption at rest, tenant data isolation, full audit logs - designed for industrial enterprise requirements from day one.

AgriTech · GTM Capstone

AgriBridge - GTM for Smallholder Farmers

WhatsApp-first marketplace. Fair price transparency. No middlemen.

Market context

110 million smallholder farmers, $500B agricultural production value, $20B Indian agtech market. Farmers receiving only 60-65% of end-market value via traditional agent routes. e-NAM reaching 17.7M farmers but still constrained by digital literacy and connectivity.

Strategic positioning

WhatsApp-first (zero learning curve), vernacular-first (Marathi, Hindi, local languages), hyper-local advisory, verified buyer listings. Competitive gap vs DeHaat (complex UI), Gramophone (limited buyer network), AgroStar (input-focused).

GTM challenges addressed

  • Digital literacy barrier - WhatsApp removes app adoption friction
  • Trust deficit - verified buyers + transparent pricing
  • Remote outreach - community-led onboarding via FPOs, local Krishi Mitra
  • Regulatory risk - WhatsApp dependency acknowledged as threat

Value proposition

Farmers earn 85-90% of end-market value vs 60-65% via traditional routes - a 25-30 percentage point income improvement for India's most underserved economic segment.

HealthTech · PM Capstone

CareLoop - Digital Chronic Disease Management

HIPAA-compliant chronic care platform connecting patients, wearables, and providers.

Problem framing

60-80M chronic patients in India (diabetes/hypertension) with fragmented care: manual health logs, infrequent doctor visits (every 3 months), no real-time provider visibility. Low data integration between patient and provider.

Discovery approach

20+ patient interviews, 5 expert consultations, prototype testing (low-fidelity UI). Two primary personas: Rajesh (52, bank manager, diabetes + BP, forgets logs, overwhelmed) and Dr. Meera Kamat (diabetologist, 200+ patients, needs AI risk prediction).

MVP core features

  • Wearable integration (BP, glucose, vitals)
  • Provider dashboard with abnormal value alerts
  • Secure messaging and basic telehealth
  • Daily health recommendations

Scale path

Phase 1: 100-200 patients pilot. Phase 2: 10+ clinics, hospital network tie-ups, 5K+ active users. Phase 3: AI-driven risk prediction, insurance integration, national expansion.
